Payment Transfer Status API

Welcome to the Zenobia Pay Transfer Status service. This service provides real-time tracking of payment transfers across our payment network.

Key Features

  • Real-time transfer status updates
  • WebSocket connections for live monitoring
  • RESTful API for status updates
  • Persistent status storage

API Documentation

POST /create

Create a new transfer with an initial status. Used to initialize a transfer before processing.

Request body: { id, status?, details? }

Initial status: "PENDING" (default) or "PROCESSING"

GET /status?id=<transferId>

Check the status of a transfer by ID. If transfer is still in progress, connect to the WebSocket for real-time updates.

Response: { id, status, details, updatedAt, websocket_url }

POST /update

Update the status of a transfer. Used by the core payment system to communicate status changes.

Request body: { id, status, details }

Status values: "PENDING", "PROCESSING", "COMPLETED", "FAILED"

GET /transfers

List all transfers in the system with their current status.

Response: Array of transfer objects

GET WebSocket: /transfers/<transferId>

Connect to this WebSocket endpoint to receive real-time updates for a specific transfer.

The connection will close automatically after 10 seconds if no update is received, or immediately upon receiving a terminal status (COMPLETED/FAILED).
